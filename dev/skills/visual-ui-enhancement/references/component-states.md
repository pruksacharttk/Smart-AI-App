# Component States

Every production UI should account for relevant states.

## Async states

- Loading: skeletons or disabled pending action.
- Empty: explain what is missing and give next action.
- Error: explain what happened and how to retry or recover.
- Success: confirm completion and next step.

## Interaction states

- Hover.
- Active/pressed.
- Focus-visible.
- Selected/current.
- Disabled.
- Invalid.

## Forms

- Field-level error.
- Form-level error.
- Submitting state.
- Save success state.
- Dirty/unsaved changes if relevant.
