# UX Review Checklist

- Is the primary action clear within 3 seconds?
- Does the page title match the user's task?
- Are related controls grouped together?
- Are forms asking only necessary information?
- Are labels specific and visible?
- Are helper texts useful and concise?
- Are validation errors close to the field and actionable?
- Can users recover from mistakes?
- Is there a clear empty state with a next action?
- Is loading communicated without layout jump?
- Are destructive actions protected by confirmation or undo?
- Does mobile preserve the same core task?
- Is jargon explained or avoided?
- Are success states clear enough?
