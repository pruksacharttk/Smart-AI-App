# Visual Polish Checklist

Use this before finalizing a UI.

## Hierarchy

- The primary action is visually obvious.
- The page title and section headings create a clear scan path.
- Secondary metadata is quieter than primary content.
- Key numbers or statuses have enough weight and context.

## Composition

- Layout has rhythm and intentional grouping.
- Cards are not repeated without hierarchy.
- Negative space is used to improve clarity.
- Sections have consistent vertical rhythm.

## Typography

- Display, heading, body, label, caption, and mono styles are distinct.
- Body text is readable at mobile and desktop sizes.
- Line length is comfortable.
- Letter spacing is intentional, not random.

## Color and surfaces

- Colors use design tokens.
- Accent color is used sparingly.
- Surfaces have subtle contrast.
- Dark mode remains readable.

## Premium restraint

- Avoid too many gradients, shadows, blurs, or animated effects.
- Use one memorable visual idea, not five competing ones.
